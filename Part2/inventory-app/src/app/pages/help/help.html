Help & FAQ

Find answers to frequently asked questions and troubleshooting guidance for using this inventory management system.

Frequently Asked Questions

Q: Will my inventory data be saved after I close the browser?

A: No. This application is designed to store data only during your active browser session. All inventory data will be cleared when you close the browser tab or window, as there is no server-side or local storage persistence.

Q: Why can't I add an item with an existing Item ID?

A: The Item ID must be unique for each inventory item, as it serves as the primary identifier for each record. If you receive an error that the Item ID already exists, please use a different unique ID for your new item.

Q: How do I edit or delete an item?

A: You can edit or delete items using the item's exact name. Go to the Inventory Manage page, enter the full name of the item you want to modify in the Edit or Delete section, fill in the new details for editing, and submit the action.

Q: How is the Stock Status determined?

A: Stock Status is automatically updated based on the item quantity:
- Out of Stock: Quantity = 0
- Low Stock: Quantity between 1 and 10
- In Stock: Quantity greater than 10

Q: Can I use this application on my mobile phone?

A: Yes! This application is fully responsive, designed to work seamlessly on both narrow mobile screens and wide desktop screens. All features are accessible on mobile browsers.

Troubleshooting

Issue: Buttons are not working when I click them

Solution: Ensure all required fields in the form are filled correctly. Buttons will be disabled if the form has invalid or missing inputs. Check for error messages below each input field to fix validation issues.

Issue: Item not found when editing or deleting

Solution: The item name must match exactly (case-insensitive). Double-check the spelling of the item name in the inventory list, and ensure there are no extra spaces before or after the name in the input field.

Issue: Application is not displaying correctly on mobile

Solution: Ensure your mobile browser is up to date. If the layout is broken, try refreshing the page. The application is optimized for modern mobile browsers (Chrome, Safari, Firefox, Edge).

Issue: All my data disappeared

Solution: This is expected behavior. The application does not persist data after the browser session ends. If you refreshed the page, closed the tab, or your browser crashed, all data will be reset to the initial sample data.
