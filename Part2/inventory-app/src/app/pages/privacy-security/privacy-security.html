Privacy & Security Requirements Analysis

This page outlines the key privacy and security considerations for this mobile inventory management application, aligned with the requirements of PROG2005 Programming Mobile Systems unit.

1. Core Security Requirements

Input Validation & Data Integrity

This application implements strict client-side input validation for all user inputs, including:

  • Required field checks to prevent incomplete data submission
  • Numeric field validation (quantity ≥ 0, price > 0) to avoid invalid data
  • Unique Item ID enforcement to maintain data integrity in the inventory database
  • Type safety via TypeScript, ensuring all data conforms to predefined interfaces and prevents type-related vulnerabilities

Client-Side Data Protection

As a client-side only application (no server-side interaction), all data is stored in the application's memory during the user session. Key protections include:

  • Session-only data persistence: All inventory data is cleared when the browser tab is closed, preventing unauthorized access to sensitive inventory data after the session ends
  • No local storage usage: Avoids persistent storage of sensitive business data on the user's device, reducing the risk of data leakage
  • Isolated data access: Inventory data is encapsulated within Angular services, preventing unauthorized access from external scripts

User Action Authentication

The application implements confirmation prompts for destructive actions:

  • Delete operations require explicit user confirmation before execution, preventing accidental data loss from unintended clicks
  • All data modification actions (add, edit, delete) provide immediate user feedback, ensuring users are fully aware of the outcome of their actions

2. Privacy Considerations

User Data Collection

This application does not collect, transmit, or store any personal user data to external servers or third-party services. All data entered by the user remains entirely on the client-side within the browser session.

Mobile-Specific Privacy Risks Mitigation

For mobile device users, the application addresses key mobile privacy risks:

  • Responsive design ensures the application does not request unnecessary device permissions (e.g., camera, location, storage), minimizing privacy exposure
  • No background data processing, ensuring the application only operates when actively used by the user
  • Compatible with mobile browser privacy features, including incognito/private browsing modes for enhanced session privacy

3. Ongoing Security Best Practices

  • Regular dependency updates: Angular and TypeScript dependencies are kept up to date to patch known security vulnerabilities
  • Secure coding practices: All code follows type-safe development practices, avoiding common client-side vulnerabilities like XSS (Cross-Site Scripting)
  • User education: The help page provides guidance for users on secure usage of the application, including not sharing sensitive inventory data in unsecure environments
